Constructor takes in the necessary state, action, reward objects and sets the parameters of the replay buffer. Currently only even sampling is supported by this replay buffer (no weighted transitions) but it could be added fairly easily.

Parameters

  • buffer_size (int) – Size of the deque before oldest entry is deleted.

  • no_delete (bool) – Set if you do not wish old entries to be deleted.

  • state (State()) – State objecct.

  • action (Action()) – action objecct.

  • reward (Reward()) – Reward objecct.

load_buffer_JSON(file_path: str)

This method should be called by the user to load the replay buffer after initialization with a Json file generated by RecordDataJSON() with the save_all paramater = True. This will populate the buffer with every timestep from every episode recorded.

Parameters

file_path (str) – file path to json file.

load_buffer_BUFFER(file_path: str)

This method should be called by the user to load the replay buffer after initialization with a previously saved replay buffer json file generated by ReplayBufferDefault. This will populate the buffer with every timestep.

Parameters

file_path (str) – file path to json file.
