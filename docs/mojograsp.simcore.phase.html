mojograsp.simcore.phase module

class Phase

Bases: ABC

Phase Abstract Base Class

abstract __init__()

Constructor should be used to pass any other objects or data that the Phase class will need to use. Usually this will be a manipulator or other mojograsp objects.

abstract setup()

This method is called by SimManager() once before the phase begins.

abstract execute_action()

This method is called every step by SimManager() while the currrent phase is active. Operations such as moving joints and executing actions should be done here.

abstract exit_condition() → bool

This method is called by SimManager() every Phase iteration and returns either a Boolean, signifying if the current phase should end or continue looping. Returning True signifies that the current phase should end and function next_phase() should be called. Any exit conditions for a phase should be kept here.

Returns

True (exit phase) or False (continue phase).

Return type

bool

abstract next_phase() → str

This method is called by SimManager() once a phase has returned True for its exit_condition(). Returns a string specifying the next phase that the PhaseManager() should transition to next. To signify the end of an episode return None.

Returns

The name of the next phase or None

Return type

str or None

pre_step()

This method is called before every step by SimManager() while the currrent phase is active. This is not an abstract method and does not need to be overidden.

post_step()

This method is called after every step by SimManager() while the currrent phase is active. This is not an abstract method and does not need to be overidden.
